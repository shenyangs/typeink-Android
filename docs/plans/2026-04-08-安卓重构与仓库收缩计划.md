# 安卓重构与仓库收缩实施计划

> **给后续实现者：** 先按本计划做“仓库收缩与依赖解耦”，再进入 UI/状态机重构。删除动作必须分阶段执行，且每阶段都要先验证 Android 核心链路。

**Goal:** 把仓库收缩为“以 Android 输入法客户端为核心”的单项目结构，并在不破坏现有录音、识别、润色、写回能力的前提下，为下一轮输入法壳与状态机重构清路。

**Architecture:** 先做“边界收缩”，把 Android 工程从根目录 Python/Web 调试资产中解耦；再做“状态单源化”，把旧的控制面板式 UI 改造成单画布输入法壳；最后再推进本地 ASR / 本地润色替换。全程保持 DashScope 现有链路作为过渡保底，避免核心能力回归。

**Tech Stack:** Android 原生（Kotlin / View 过渡到新渲染层）、InputMethodService、Gradle、现有 DashScope 在线识别与润色链路、项目内工程文档。

---

## 一、目标边界

### 1. 本轮要实现什么

1. 仓库目录收缩为“Android 客户端优先”的结构；
2. 移除仅用于本地功能验证的 Python / Web / Vercel / 假服务脚本；
3. 保留 Android 当前能工作的核心功能：
   - APK 可构建；
   - 输入法可启用、可切换；
   - 麦克风可申请权限；
   - 语音开始/停止链路可走通；
   - 识别结果可进入润色并写回；
   - “修最近一句”和手动键盘 fallback 不因仓库收缩失效；
4. 为后续“隐形 UI + 单状态源 + 单路径输出”重构腾出干净边界。

### 2. 本轮明确不做什么

1. 不在同一轮里同时切本地 Qwen3-ASR；
2. 不在仓库收缩时同步大改输入法 UI；
3. 不在删除旧目录前直接改掉所有历史文档，只先保留安卓仍需要的文档主线。

## 三、重构阶段计划

### 阶段 0：建立安全基线

**目标：** 在任何删除和重构前，先固定“什么叫核心功能没坏”。

**文件：**
- 保留并使用：`android/app/src/main/...`
- 更新：`docs/系统架构与进度.md`

**步骤：**
1. 固定最小验收清单：
   - `:app:assembleDebug`
   - `:app:testDebugUnitTest`
   - 真机或模拟器手测 6 项：启用输入法、切换输入法、开始录音、停止录音、结果写回、修最近一句。
2. 把这 6 项写进文档，后续每阶段都复验。
3. 删除前先保留一份可回退分支或标签。

**通过标准：**
- Android 构建和单测通过；
- 删除前后的验收标准完全一致。

### 阶段 1：先把 Android 从根目录非安卓依赖中解开

**目标：** 删除前先消除 Android 对仓库根目录遗留资产的依赖。

**文件：**
- Modify: `android/app/build.gradle`
- Modify: `README.md`
- Modify: `docs/系统架构与进度.md`
- Optional Create: `android/app/src/main/assets/`

**步骤：**
1. 清查 `app/static/*` 是否被 Android 运行时真正使用。
2. 如果未使用：
   - 从 `android/app/build.gradle` 去掉 `assets.srcDirs = ["../../app/static"]`。
3. 如果仍需保留其中某个 HTML 原型：
   - 迁入 `android/app/src/main/assets/归档原型/`，不要继续跨目录引用。
4. 重新梳理密钥来源：
   - 优先改为 `android/local.properties` 或 Android 工程内的本地配置文件读取；
   - 至少让 Android 不再默认依赖仓库根目录 `.env.local`。
5. 更新 README，去掉 Python/Web/Vercel 说明，只保留 Android 构建与运行说明。

**通过标准：**
- Android 构建不再依赖根目录 `app/`；
- Android 构建路径解释清晰，不需要 Python 服务端上下文。

### 阶段 2：执行仓库收缩

**目标：** 真正把非安卓验证资产从仓库中移除。

**文件：**
- Delete: 上文“A. 立即可删”全部内容
- Delete: `app/`（在阶段 1 完成后）
- Modify: `docs/系统架构与进度.md`
- Modify: `README.md`

**步骤：**
1. 先删除纯 Python / Web / Vercel / 假服务资产。
2. 再删除 `app/` 整包。
3. 清理 README 中所有失效命令、目录树和部署说明。
4. 清理与后端原型强绑定、且不再服务 Android 的文档。

**通过标准：**
- 仓库顶层只剩 Android 主体、必要文档、必要构建脚本和输出目录；
- 删除后 `assembleDebug`、`testDebugUnitTest` 仍通过。

### 阶段 3：重建 Android 内部目录边界

**目标：** 让 Android 客户端从“原型堆叠”变成“可持续重构”的结构。

**建议目录：**
- `com.typeink.ime.core`
- `com.typeink.ime.session`
- `com.typeink.ime.pipeline`
- `com.typeink.ime.ui`
- `com.typeink.ime.fallback`
- `com.typeink.ime.platform`

**步骤：**
1. 把 `prototype/` 与 `inputmethod/` 中的职责重新拆分。
2. 把下面三类对象分开：
   - 平台层：IME、权限、InputConnection、Insets；
   - 会话层：录音、ASR、润色、提交；
   - 渲染层：输入法画布、状态渲染、手动键盘 fallback。
3. 明确哪些类只是过渡遗产：
   - `MainActivity`
   - `EditPanelActivity`
   - `TypeinkInputView`
   - `ImmersiveEditorActivity`

**通过标准：**
- 目录结构能一眼看出“平台 / 会话 / UI”的职责；
- 后续 UI 重写时不用再跨文件硬找状态。

### 阶段 4：建立单一状态源

**目标：** 从“多个布尔值 + 多块文本框 + visibility 切换”改为“单状态源驱动单画布”。

**核心状态建议：**
- `Idle`
- `Priming`
- `Listening`
- `Polishing`
- `Committed`
- `Editing`
- `Error`

**状态数据建议：**
- `displayText`
- `committedText`
- `partialText`
- `amplitude`
- `isFallbackKeyboardVisible`
- `editContext`
- `recentCommit`

**步骤：**
1. 停止让 `TypeinkInputView` 自己维护 `isRecording / isEditMode / isProcessingEdit / currentMode`。
2. 让 IME 只有一个状态仓库。
3. 让 ASR partial、LLM delta、最终写回都只改同一个 `displayText`。
4. 把“草稿文本”和“润色文本”的双轨 LiveData 收敛成单轨展示模型。

**通过标准：**
- UI 不再需要维护 `draftText` 和 `finalText` 两套视觉出口；
- 任意时刻都能用一个状态对象还原当前界面。

### 阶段 5：重做输入法画布，但不先动核心能力

**目标：** 先换壳，不换心脏。

**新 UI 树建议：**
- `TypeinkImeRoot`
- `SystemAwareGlassCanvas`
- `ThoughtTextRegion`
- `VoiceOrb`
- `ContextMicroActions`
- `FallbackKeyboardSheet`
- `TransientFeedbackLayer`

**步骤：**
1. 保留现有 DashScope 识别/润色链路作为过渡；
2. 新画布只接一个状态源；
3. 默认只显示：
   - 单文本区
   - 语音光球
   - 极少量上下文操作
4. 传统键盘不再与主面板并列常驻，只在用户手动修字时上浮显示。

**通过标准：**
- 输入法默认形态不再像控制面板；
- 用户只看到“正在成文的结果”，不被迫理解“草稿 vs 润色”的内部过程。

### 阶段 6：再做离线化替换

**目标：** 在新壳稳定后，把云端能力逐步替换为本地能力。

**步骤：**
1. 先定义统一 `SpeechPipeline` 接口；
2. 保留 `DashScopePipeline` 作为过渡实现；
3. 新增真正的 `LocalQwenAsrPipeline`；
4. 最后再评估本地润色能力是否同样落端。

**通过标准：**
- 离线与在线只是在 pipeline 实现上切换；
- UI 层完全无感知。

## 四、核心功能不回归策略

每阶段都必须复验以下项目：

1. `cd android && ./gradlew :app:assembleDebug`
2. `cd android && ./gradlew :app:testDebugUnitTest`
3. 启用输入法
4. 切换到 Typeink
5. 点击录音开始
6. 点击停止后拿到最终文本
7. 文本成功写回输入框
8. “修最近一句”仍可触发
9. 手动键盘 fallback 仍能打开与收起

## 五、建议的执行顺序

推荐先做这 3 个动作，不要乱序：

1. 先解耦 Android 对根目录 `app/` 和 `.env.local` 的依赖；
2. 再执行仓库删除；
3. 再开始 UI / 状态机重构。

## 六、当前建议

如果用户同意删除，我建议先做一个“保守收缩版”：

1. 先删所有 Python / Vercel / 测试客户端 / 假服务脚本；
2. 暂时保留 `README.md`、Android 文档、`tools/build_apk.sh`、`.env.local.example`；
3. `app/` 等到 Android 彻底解耦后再删；
4. 这能最大程度降低“刚删完仓库，Android 反而编不过”的风险。
